AWS 사용하기 - VPC의 개념

2018-06-09
aws

AWS 서비스를 사용하다보면 VPC와 Subnet을 지정해줘야 하는 경우가 많다. 간단하게 개념을 정리했다.

VPC 기본 원리를 설명한 유튜브 영상

VPC : Amazon Virtual Private Cloud

VPC는 사용자의 AWS 계정 전용 가상네트워크다. EC2 인스턴스와 같은 AWS리소스를 VPC에서 실행할 수 있다.

  • AWS를 생성하면 디폴트 VPC가 자동으로 생성된다.
  • VPC를 사용하는데 필요한 기본적인 사항들이 함께 생성된다.(아래 목록)
    • 1) Internet Gateway(IGW) (모뎀같은 역할)
    • 2) A route table (라우터/스위치 같은 역할)
    • 3) A network access control list(NACL) (방화벽 같은 역할)
    • 4) Subnets to provision AWS resources in(EC2 같은)

서브넷(Subnet)

서브넷은 VPC의 IP 주소 범위. 네트워크의 일부다.

  • 지정된 서브넷으로 AWS 리소스를 시작할 수 있다.
  • 서브넷은 라우팅 테이블과 연결돼있어야 함.
  • 퍼블릭 vs 프라이빗
    • 퍼블릭 : 인터넷에 연결돼있음. = IGW에 연결된 route table에 연결돼있음.
    • 프라이빗: 인터넷에 연결돼있지 않음 = IGW에 연결되지 않은 route table에만 연결돼있음
  • 서브넷이 프라이빗인지 퍼블릭인지 결정하는 건 서브넷 자체에서가 아니라, 연결된 route table이 IGW(internet gateway)에 연결돼있느냐 아니냐임.
  • 서브넷은 한개 AZ 안에만 있을 수 있음. (vpc는 여러 Availability Zone에 걸쳐 있음)
